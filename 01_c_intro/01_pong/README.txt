Реализация консольного pong

--------------------------------------------------------------------------------
Самый простой вариант - реализовать пошаговый pong. А затем, если хочется,
реализовать в реальном времени.
Если получится, то можно ещё написать что-то типа ИИ для второго игрока.

Определиться с клавишами управления.
Определиться с размерами экрана. 80 x 25 пойдёт.
(Можно сделать поле динамическим - чтобы оно масштабировалось по размеру терминала)

Размер ракетки - пускай 3 символа.
Шар - 1 символ

Если делать в пошаговом режиме - пригодится какая-то клавиша для пропуска игрового тика, например, пробел.

Если делать в реальном времени - копать в направлении non-canonical terminal imput.

Структура проекта - по классике

          Makefile
          src/
          out/
